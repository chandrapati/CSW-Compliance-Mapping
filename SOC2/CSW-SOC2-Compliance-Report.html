Executive Summary

SOC 2 Type II audits evaluate whether security controls operated effectively over a defined audit period (typically 6–12 months). Cisco Secure Workload (CSW) directly satisfies the Security Trust Services Criteria and contributes to Availability, Confidentiality, and Processing Integrity criteria through continuous workload visibility, micro-segmentation, vulnerability detection, and forensic telemetry.

Key CSW Capabilities for SOC 2

CC6 — Logical & Physical Access Controls

CSW enforces workload-level allowlist policies that satisfy CC6.1 and CC6.6. Every inbound connection to in-scope systems requires explicit policy approval. ADM generates a complete, auditable map of all communication paths, eliminating undocumented access routes that commonly fail SOC 2 audits.

CC7 — System Operations

Continuous CVE scanning with CVSS scoring satisfies CC7.1 vulnerability detection requirements. Full process and network telemetry with anomaly detection satisfies CC7.2 system monitoring. Forensic telemetry with flow search enables the security event evaluation required by CC7.3.

CC8 — Change Management

CSW policy workspace versioning provides an immutable audit trail of all security control changes. Quarterly ADM re-runs detect unauthorized changes to system communication patterns — serving as continuous configuration drift detection for CC8.1.

Deployment Recommendations

  • Enable full flow telemetry retention from Day 1 — SOC 2 Type II requires evidence across the entire audit period (6–12 months minimum)

  • Configure alert export to SIEM for centralized evidence collection and automated review

  • Run ADM before enforcement — document the approved baseline before locking down communication paths

  • Schedule quarterly ADM re-runs to detect configuration drift for CC8.1 change management evidence

  • Set CVSS ≥ 7.0 alert threshold for immediate notification on high-severity vulnerabilities
