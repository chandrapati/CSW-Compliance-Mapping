Executive Summary

The CISA Zero Trust Maturity Model (ZTMM) v2.0 defines maturity across five pillars: Identity, Devices, Networks, Applications & Workloads, and Data. Each pillar progresses through four stages: Traditional → Initial → Advanced → Optimal. Cisco Secure Workload (CSW) is the primary platform for achieving Advanced-to-Optimal maturity in the Networks pillar and significant progress in the Applications & Workloads and Data pillars.

CSW's workload-level micro-segmentation, continuous ADM, and forensic telemetry align directly with CISA's emphasis on continuous validation, least-privilege access, and dynamic policy enforcement — the hallmarks of Optimal maturity.

ZTMM Deployment Roadmap

Phase 1 — Traditional to Initial (Weeks 1–4)

  • Deploy CSW sensors on all on-prem workloads; connect cloud accounts via CSW cloud connectors

  • Run Application Dependency Mapping (ADM) to establish complete traffic baseline

  • Define scope hierarchy aligned to ZTMM pillar boundaries

  • Enable flow telemetry retention (minimum 12 months)

Phase 2 — Initial to Advanced (Weeks 5–12)

  • Build allowlist policies from ADM baseline — enforce least-privilege network access

  • Run simulation mode for 2 weeks; validate no false positives before enforcement

  • Enable enforcement mode on sensitive scopes first; expand progressively

  • Detect and block non-encrypted flows; enforce HTTPS/SFTP/LDAPS across all scopes

  • Configure anomaly detection and SIEM integration for continuous monitoring

Phase 3 — Advanced to Optimal (Month 4+)

  • Implement 90-day ADM re-run cycle for continuous policy validation

  • Integrate CSW API with SOAR for automated policy response to incidents

  • Enable vulnerability-triggered policy review — critical CVEs tighten access automatically

  • Conduct quarterly ZTMM maturity assessment against scorecard
